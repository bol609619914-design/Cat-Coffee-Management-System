# 更新日志

本文档用于记录项目的重要迭代与阶段性交付内容。

## [2026-04-22] 第二期会员营销与业务中台扩展

### 新增

- 新增会员营销中心后端模块，统一承载积分、优惠券、评价、活动营销等业务能力。
- 新增会员积分体系，包含 `sys_user.member_points` 余额字段与 `member_point_flow` 积分流水表。
- 新增优惠券能力，包含 `coupon_template` 优惠券模板表与 `user_coupon` 用户优惠券表。
- 新增订单评价能力，包含 `order_review` 评价表，并支持按订单饮品项归属评价。
- 新增活动营销能力，包含 `marketing_activity` 活动主表与 `marketing_activity_rule` 活动规则表。
- 新增营销接口控制器与服务层，实现积分查询、优惠券模板管理、用户领券、后台发券、评价管理、活动管理等接口。
- 新增普通用户“积分中心”“我的优惠券”“我的评价”“活动专区”专属视图。
- 新增管理员 / 店员营销运营入口，可查看积分流水、优惠券模板、活动规则与评价数据。

### 优化

- 优化订单业务结构，为订单补充原价、优惠金额、应付金额、使用积分、奖励积分、关联优惠券等字段。
- 优化会员权益联动逻辑，使订单下单过程支持积分抵扣与优惠券选择。
- 优化登录用户信息返回体，在用户资料中补充会员积分字段，前端可直接展示当前余额。
- 优化菜单与首页跳转策略，按用户权限自动进入会员中心或后台首页。
- 优化数据库初始化脚本，使全量建库时即可直接获得第二期营销相关表结构、权限与示例数据。

### 修复

- 修复普通用户预约时间提交的后端反序列化异常，统一预约时间格式为 `yyyy-MM-dd HH:mm:ss`。
- 修复后端运行时数据库升级缺少营销字段的问题，启动时可自动补齐积分、优惠券、活动相关结构。
- 修复订单模型仅支持基础金额字段的问题，避免后续接入会员优惠时数据表达不完整。

### 说明

- 本次迭代属于“完整业务中台扩展”而非简单补丁，已覆盖积分发放、积分抵扣、优惠券领取/发放/核销、评价归属、活动有效期与规则配置等关键能力。
- 当前仓库已具备课程设计、毕业设计、项目答辩展示所需的核心会员营销闭环；后续可继续向积分商城、满减策略引擎、活动报名、消息通知等方向扩展。

## [2026-04-21] 第一期功能交付

### 新增

- 新增猫咪头像与饮品图片上传能力。
- 新增文件存储抽象层，并提供本地存储实现，为后续切换阿里云 OSS、MinIO、腾讯云 COS 等对象存储预留扩展位。
- 新增上传文件静态资源映射，支持前端直接预览上传后的图片。
- 新增预约与订单的用户归属字段，实现普通用户数据隔离。
- 新增数据库自动升级逻辑，可自动补齐 `reservation.user_id` 与 `customer_order.user_id` 字段及索引。
- 新增基于 ECharts 的经营分析图表，包括近 7 天营收趋势与订单状态分布。
- 新增登录页品牌 Logo 与浏览器标签页 favicon。
- 新增项目级 `CHANGELOG.md`，用于版本变更追踪。

### 优化

- 优化登录页视觉层级，加入品牌 Logo、品牌副标题，并重构登录 / 注册切换区域样式。
- 优化侧边栏品牌区，使用自定义猫咖 Logo 替代原有文字占位块。
- 优化侧边栏菜单交互，统一悬停态与选中态的圆角视觉风格。
- 优化侧边栏底部操作区，将“修改密码”调整到“退出登录”上方。
- 优化普通用户菜单文案，将后台语义改为“我的预约”“我的订单”等前台视角表达。
- 优化普通用户预约与下单表单，隐藏不必要的后台管理字段。
- 优化前端按钮、抽屉表单、品牌展示与局部交互样式，提升整体一致性。
- 优化 README 展示截图，更新为当前版本最新界面效果。

### 修复

- 修复上传图片后前端预览 `FAILED` 的问题，原因是静态资源映射路径未正确生效。
- 修复前端上传请求头处理不稳定的问题，改为交由浏览器自动生成 multipart 边界信息。
- 修复普通用户仍可看到非本人预约与订单的问题，接口层已改为按当前登录用户过滤。
- 修复初始化数据未正确绑定普通用户导致“我的预约 / 我的订单”为空的问题。
- 修复侧边栏悬停态与选中态形状不一致的问题。
- 修复 Logo 显示中的白底与裁切问题，改为透明底展示与更自然的投影效果。

### 说明

- `backend/uploads/` 为运行时上传目录，现已通过 `.gitignore` 排除，不再进入版本库。
- 当前本地存储实现适合课程设计、毕业设计、本地演示与原型交付；如果后续需要部署上线，可直接基于现有接口切换为正式对象存储方案。
